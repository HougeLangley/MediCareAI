## 📮 邮件服务商配置指南 | Email Provider Configuration

### QQ邮箱（推荐国内用户）| QQ Mail (Recommended for China)

**QQ邮箱完全支持！** 但需要特殊配置：

#### 配置步骤：

1. **开启SMTP服务**
   - 登录 QQ邮箱网页版: https://mail.qq.com
   - 点击"设置" → "账户"
   - 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
   - 开启"SMTP服务"
   - 会收到一个**授权码**（16位字符串），请保存好！

2. **在 MediCareAI 中配置**
   ```
   SMTP服务器: smtp.qq.com
   端口: 465（推荐）或 587
   用户名: your-qq@qq.com
   密码: 授权码（不是QQ登录密码！）
   发件人邮箱: your-qq@qq.com
   发件人名称: MediCareAI
   使用TLS: 是
   ```

3. **测试发送**
   - 点击"测试"按钮
   - 输入测试邮箱地址（可以是另一个QQ邮箱）
   - 检查收件箱和垃圾箱

#### QQ邮箱限制：
- 发件频率限制：每分钟不超过10封
- 每天发送上限：约100封（新账号可能更低）
- 必须使用授权码，QQ密码无法通过SMTP验证

#### 常见问题：

**问题1：提示"授权码错误"**
- 确保使用的是16位授权码，不是QQ密码
- 重新在QQ邮箱设置中生成新的授权码

**问题2：邮件进入垃圾箱**
- 这是正常现象，收件人需要添加发件人到白名单
- 建议在邮件正文中提醒用户检查垃圾箱

**问题3：连接超时**
- 尝试将端口从587改为465
- 确保服务器能访问 smtp.qq.com:465

## 🔧 故障排除 | Troubleshooting

## 🔧 故障排除 | Troubleshooting

### 问题1：邮件发送失败

**症状：** 测试邮件发送失败，日志显示连接超时

**解决方案：**
1. 检查 SMTP 服务器地址和端口是否正确
2. 确认防火墙允许出站 587/465 端口
3. 检查用户名和密码是否正确（使用应用专用密码）
4. 查看邮件服务商的安全设置

### 问题2：配置不生效

**症状：** 修改配置后邮件仍使用旧配置发送

**解决方案：**
1. 确认配置已设为"默认"
2. 重启后端服务
3. 检查 `email_configurations` 表中的 `is_default` 字段

### 问题3：数据库迁移失败

**症状：** 运行迁移时出错

**解决方案：**
1. 检查数据库连接是否正常
2. 确认 `users` 表已存在（外键依赖）
3. 手动检查表是否已存在：
   ```sql
   SELECT * FROM information_schema.tables 
   WHERE table_name = 'email_configurations';
   ```
